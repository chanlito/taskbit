# Agent Instructions

## Package Manager

- Use **pnpm** only: `pnpm install`, `pnpm dev`, `pnpm build`, `pnpm test`
- Do not add `package-lock.json`; keep `pnpm-lock.yaml` authoritative

## Workspace Map

| Path              | Owns                                                                                 |
| ----------------- | ------------------------------------------------------------------------------------ |
| `apps/web`        | React 19 + Vite + TanStack Router SPA on `http://localhost:6900`                     |
| `apps/api`        | Nitro API routes on `http://localhost:6969`, including `GET /` and `GET /api/health` |
| `apps/desktop`    | Tauri v2 wrapper that reuses the web app build/dev server                            |
| `packages/shared` | Shared constants and TypeScript API contracts                                        |

## Root Commands

Use these for final verification or cross-workspace changes, not after every edit.

| Task            | Command                        |
| --------------- | ------------------------------ |
| Web + API dev   | `pnpm dev`                     |
| Web only        | `pnpm dev:web`                 |
| API only        | `pnpm dev:api`                 |
| Desktop         | `pnpm dev:desktop`             |
| Desktop install | `pnpm build:desktop:install`   |
| Build           | `pnpm build`                   |
| Push build      | `pnpm build:push`              |
| Test            | `pnpm test`                    |
| Lint            | `pnpm lint`                    |
| Format check    | `pnpm format`                  |
| Typecheck       | `pnpm typecheck`               |

## File-Scoped Commands

| Task                 | Command                                                              |
| -------------------- | -------------------------------------------------------------------- |
| Web route lint       | `pnpm --filter @taskbit/web exec oxlint src/routes/_marketing.index.tsx` |
| Web route test       | `pnpm --filter @taskbit/web test -- src/routes/-index.test.tsx`      |
| Web route format     | `pnpm --filter @taskbit/web exec oxfmt --check src/routes/_marketing.index.tsx` |
| API route test       | `pnpm --filter @taskbit/api test -- server/api/health.test.ts`       |
| Shared contract test | `pnpm --filter @taskbit/shared test -- src/index.test.ts`            |
| Desktop Rust check   | `cargo check --manifest-path apps/desktop/src-tauri/Cargo.toml`      |

## Development Workflow

- Prefer the narrowest affected workspace or file-scoped command while iterating
- Do not run root `pnpm build`, `pnpm test`, `pnpm lint`, or `pnpm typecheck` after every edit
- `pnpm prepush` uses `pnpm build:push`, which excludes the Tauri desktop package build
- Run root checks when changes cross workspaces, touch shared config/dependencies, or before handing off complete work

## Key Conventions

- Keep app code in workspace-local `src/`; keep web static assets in `apps/web/public/`
- Pages in `apps/web/src/routes/` must define a document title via TanStack Router `head()`
- Never edit `apps/web/src/routeTree.gen.ts`; it is generated by the TanStack Router plugin
- Put Nitro API routes under `apps/api/server/api/` and non-API routes under `apps/api/server/routes/`
- Keep Nitro route tests beside the route file and make sure `nitro.config.ts` ignores `*.test.ts`
- Import shared payload types from `@taskbit/shared`
- Desktop must reuse `apps/web`; do not create a second frontend inside `apps/desktop`
- Keep dependency ownership local to the workspace that uses it; use `workspace:*` for internal packages
- Do not commit generated local state: `dist/`, `.output/`, `.tanstack/`, `.nitro/`, or Tauri `target/`
- Do not add commit-attribution requirements to this project

## Commit Messages

- Use conventional commits: `type(scope?): emoji msg`
- Example: `feat(web): ✨ add about route`
