# Agent Instructions

## Package Manager

- Use `npm`: `npm install`, `npm run dev`, `npm run build`, `npm run test`

## File-Scoped Commands

| Task         | Command                                            |
| ------------ | -------------------------------------------------- |
| Lint         | `npx oxlint src/routes/index.tsx`                  |
| Test         | `npm run test -- src/routes/-index.test.tsx`       |
| Format check | `npx oxfmt --check src/routes/index.tsx`           |
| Format write | `npx oxfmt src/routes/index.tsx`                   |
| Typecheck    | `npx tsc --noEmit --pretty false -p tsconfig.json` |

## Coding Workflow

- Prefer targeted checks (specific files, relevant tests) over repo-wide commands during development

## Key Conventions

- Keep app code in `src/`; keep static assets in `public/`
- Put route files in `src/routes/`; keep router setup in `src/router.tsx`, bootstrapping in `src/main.tsx`, and global styles in `src/styles.css`
- Pages in `src/routes/` must define a document title via TanStack Router `head()` metadata
- Never edit `src/routeTree.gen.ts`; it is generated by the TanStack router plugin
- Use TypeScript, React function components, and ES modules
- This project uses React Compiler via `reactCompilerPreset()` in `vite.config.ts`
- Prefer Tailwind utilities before expanding `src/styles.css`
- Use `#/*` or `@/*` aliases for `src/` imports and `tests/*` for shared test infrastructure
- Name route files by URL segment such as `src/routes/index.tsx` or `src/routes/about.tsx`
- Use kebab-case for component filenames
- Keep app tests beside the code they cover in `src/`; keep repo/tooling tests and shared helpers in the root `tests/` folder
- Do not commit generated local state such as `dist/` or `.tanstack/`
- After adding routes, confirm `src/routeTree.gen.ts` updates cleanly and the app still boots
- Use short imperative commit subjects such as `Add about route` or `Refine home layout`

## Commit Messages

- Use conventional commits: `type(scope?): emoji msg`
- Example: `feat(auth): ✨ add login route`
